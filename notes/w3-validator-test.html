W3 validator math sanity check

Reproduces the inline validator comparison for MultiTokenDepositModal.validateDeposit / MultiTokenWithdrawModal.validateWithdrawal / MultiTokenTransferModal.validateTransfer. The validator now reads balance := BigInt(token.balance) instead of balance := convertToWei(token.balance, token.decimals) because token.balance is already a raw-wei decimal-integer string at every call site in useVault.ts.

A page-level amount like "1.5" is a human-decimal string (the user typed it). A page-level balance like "1000000000000000000" is a raw-wei integer string (it comes from BigInt.toString()). The fix's whole point: don't treat those two strings the same way.

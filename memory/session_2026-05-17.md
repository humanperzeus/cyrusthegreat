  location hardhat uses; no source duplication)
- `tools/foundry-tests/test/` for test files

  — submodules don't play with git worktrees, vendoring sidesteps that)

**27 tests written, all passing** in ~5ms total CPU time. Coverage:
- 5 constructor-validation revert paths
- 4 view assertions (currentEpoch, EPOCH_LENGTH, getPoolBucketSize, zkVerifier)
- 6 commitToPool tests (ETH + ERC-20 happy paths; reuse, bad bucket,
  wrong msg.value reverts; event-has-no-msg.sender)
- 8 revealFromPool tests (ETH + ERC-20 happy; same-epoch, unknown, MEV-
  redirect, double-spend, zero-withdrawTo, wrong-bucket reverts)

Tests use `vm.warp()` to skip across epoch boundaries — no real-world
wait. Foundry catches regressions in ~5ms; debug-UI on-chain testing
catches the wallet/RPC/chain edge cases Foundry's in-memory EVM doesn't.

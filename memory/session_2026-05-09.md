## Decisions / shipped / verified

### Live-site recovery — multi-stage incident, all resolved

1. **EIP-55 checksum issue** (root cause of "Failed to get fresh fee"). Bank8 Sepolia address `0xb83A814097C70DB79568b663662eA07e77D4D87a` had wrong-cased `D` (canonical EIP-55 has lowercase `d`). Survived months under viem 2.36 (forgiving) but surfaced when yesterday's lockfile regen pulled in viem 2.48 (strict). **Fix:** canonicalized via `ethers.getAddress(...)` in all addresses everywhere; recorded as **L-007** in tech_learnings.md.

2. **API keys auto-revoked by providers** (after the leak became visible). Tested all 4: Alchemy returned HTTP 403 "App is inactive" (Alchemy's GitHub-secret-scanning partnership response); Etherscan + BSCScan returned "Invalid API Key #err2"; Ankr survived (not in partnership). User rotated Alchemy + Etherscan; verified working via debug-UI API health check.

3. **Cloudflare dashboard env vars vs committed `.env.production`.** Tried committing `.env.production` with all build config (initially including API keys — that's how the leak compounded). After security review, refactored to: only non-sensitive structural config in `.env.production`, secrets in dashboard. Then user pushed back: too many places. Final design: dashboard-only for everything (no `.env.production` in repo at all). One source of truth.

4. **Bulk Cloudflare env-var sync** (`tools/cf-sync-env.sh`). Reads local `.env`, PATCHes Cloudflare Pages API in one call. Aliased as `ctg-sync-env` in `~/.zshrc`. Credentials sourced from `~/.config/cyrusthegreat/cloudflare.env` (mode 0600).

## Lessons learned this session (added to tech_learnings.md)

- **L-008**: Don't commit secrets to git "because they're already in the bundle anyway" — the GitHub-public-repo threat model is different (automated scrapers, secret-scanning partnerships that auto-revoke). Bundle exposure ≠ git exposure.
- **L-009**: When N config entries all embed the same secret (e.g. 8 RPC URLs each with the same API key in path), store the secret once and compose the entries in code. Reduces rotation pain to one config update.
